**결론**

네, 일부 공개 코드에서는 **사용자가 dose 값을 바꿀 수 있습니다.**  
하지만 **defocus는 보통 임의의 연속값을 직접 넣는 구조가 아니라, 미리 준비된 focus/defocus 광학 kernel 중 하나를 쓰는 구조**인 경우가 많습니다.

즉 가장 정확한 결론은 이겁니다:

> 하나의 타겟 레이아웃에서 여러 printed layout을 뽑는 구조는 가능하다.  
> 다만 공개 코드 기준으로는 `dose`는 config 값으로 바꿀 수 있는 경우가 있고, `defocus`는 보통 `focus kernel / defocus kernel`처럼 미리 계산된 조건만 선택 가능하다.  
> 임의의 defocus nm 값을 자유롭게 넣으려면 그 defocus에 해당하는 optical kernel을 새로 만들거나, 애초에 defocus를 입력으로 받는 simulator/model이어야 한다.

**근거 1: LithoBench는 dose corner를 config로 둠**

LithoBench GitHub README에는 simulator가 mask를 받아 **maximum, nominal, minimum corner**의 printed image를 출력한다고 되어 있습니다. 예시 코드도 이렇게 되어 있습니다.

```
printedNom, printedMax, printedMin = self.simLitho(mask)
```

그리고 `lithosimple.txt` config에는 실제로 dose 값이 들어 있습니다.

```
DoseMax 1.02
DoseMin 0.98
DoseNom 1.00
```

근거: [LithoBench README](https://github.com/shelljane/lithobench), [lithosimple.txt](https://raw.githubusercontent.com/shelljane/lithobench/main/config/lithosimple.txt)

**근거 2: LithoBench 코드상 defocus는 숫자 입력이 아니라 defocus kernel 선택**

LithoBench의 `pylitho/exact.py`를 보면 kernel을 읽을 때 `focus.pt` 또는 `defocus.pt`를 선택합니다. 그리고 `printedMin` 쪽은 defocus kernel을 사용합니다.

즉 코드 구조가:

```
nominal: DoseNom + focus kernel
max:     DoseMax + focus kernel
min:     DoseMin + defocus kernel
```

에 가깝습니다.

그래서 여기서의 defocus는 사용자가 `defocus = 37 nm`처럼 넣는 방식이 아니라, **이미 준비된 defocus kernel을 쓰는 방식**입니다.

근거: [LithoBench exact.py](https://raw.githubusercontent.com/shelljane/lithobench/main/pylitho/exact.py)

**근거 3: LithoDreamer는 process parameter로 focus/dose를 명시함**

LithoDreamer dataset 문서에는 각 샘플이 process parameter를 가진다고 되어 있고, 값도 명시되어 있습니다.

```
Focus (nm): 0, 50
Exposure dose: 1.0, 1.2
```

즉 이 데이터셋은 dose/focus 조건별 샘플을 갖는 구조입니다. 하지만 이것도 “사용자가 아무 값이나 연속적으로 바꾼다”보다는 **정해진 process configuration 조합 안에서 학습/평가**하는 쪽에 가깝습니다.

근거: [LithoDreamer DATASET.md](https://github.com/7jiangyq/lithodreamer/blob/main/DATASET.md)

**근거 4: TorchResist는 dose/defocus 직접 sweep 도구라기보다 resist simulator**

TorchResist README를 보면 litho simulation 결과를 만든 뒤 resist simulation을 수행합니다. 사용 예시는 `--lithomodel`, `--lithoresult`, `--resolution` 정도이고, README상 CLI에서 dose/defocus를 직접 sweep하는 인자는 보이지 않습니다. 다만 내부 litho model은 ICCAD13/FuILT 쪽 결과를 사용할 수 있습니다.

근거: [TorchResist README](https://github.com/ShiningSord/TorchResist), [TorchResist paper](https://arxiv.org/html/2502.06838)

**최종 판정**

당신 질문에 딱 맞게 답하면:

```
Q. 하나의 타겟 레이아웃으로 dose/defocus 바꿔 여러 printed layout 뽑을 수 있나?

A. 가능하다. LithoBench 같은 공개 코드도 nominal/max/min corner 출력을 지원한다.

Q. 내가 dose 값을 바꿀 수 있나?

A. LithoBench 기준으로는 가능하다. config의 DoseNom/DoseMax/DoseMin 값을 바꾸면 된다.

Q. 내가 defocus 값을 연속적으로 바꿀 수 있나?

A. 공개 코드 그대로는 애매하다. LithoBench는 defocus 숫자를 입력받기보다 미리 저장된 defocus kernel을 선택한다. 임의 nm defocus를 쓰려면 해당 조건의 kernel을 새로 생성해야 한다.

Q. 딥러닝 모델에서도 가능한가?

A. 모델이 layout만 입력받으면 불가. layout + dose + focus를 함께 입력받도록 학습된 conditional model이면 가능.
```

그래서 지금 단계에서 가장 방어적인 표현은:

> 공개 computational lithography 코드/데이터셋에는 dose와 defocus 개념이 실제로 들어가며, 하나의 mask/layout에 대해 process corner별 printed 결과를 생성하는 구조가 존재한다. 다만 dose는 config로 바꾸기 쉬운 반면, defocus는 보통 사전 계산된 optical kernel 또는 정해진 process configuration에 묶여 있어 임의 연속값을 바로 입력하는 구조라고 단정하면 위험하다.
